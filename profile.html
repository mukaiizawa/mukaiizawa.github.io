概要

大学では物理を専攻。大学4年時に今の職場にアルバイトとしてお世話になり、そのまま卒業と同時に就職。Common LispとVimをこよなく愛する。車輪の再発明が大好きで、日々怪しいものを作成している。形式言語界隈が好きで勉強中。仕事はWeb系だけどあまり興味はない（爆）

最近は片手間に軽量プログラミング言語Parenを作成している。

シンプルを信条にしているが、気づいたらbiggerismに屈していて職場の先輩に馬鹿にされる。

言語

C

アルバイト面接前、ほとんどプログラムとは無縁だったため、何か学んでおかなくてはと焦って勉強した言語。が、今日まで業務で使ったことはない（笑）Paren作成のため、必要に迫られて再学習した。

実は、大学在学中に単位を落としたコマが二つある。1つは、ほとんど欠席した体育。もう1つは、コンピュータ入門（C言語）だ。

Java

C言語の次に学んだ言語。Webアプリケーション等、業務で多用。趣味ではほとんど使用しない。

JavaScript

業務で使用。使用してもほとんどjQueryで、ピュアなjsはあまり使用したことはない。

Common Lisp

最も好きな言語。アルバイト中にJavaの次に独習した。趣味で多用しており、このブログもS式から生成されている。ピュアなCommon Lispに、自作のライブラリを使用することが多い。そのため、パッケージマネージャーやライブラリには暗い。

VimScript

Vimが好きなのでほんの少し読み書き可能。生粋のVimScript職人の方々には常々脱帽。

Paren

現在作成中。学習コストが低く、シンプルなLL言語を目指している。

VBA

仕事で何かと使う機会が多い言語。業務を効率化しようと試行錯誤するたびになんとなく読み書き可能になった。

Ruby

友達のECサイトをRuby on Railsで作成する手伝いをしたときに学んだ。

その他

Haskellに興味があるけどなかなか学べていない。

  • PL/0

好きな本

計算機プログラムの構造と解釈

計算機科学の古典的教科書。Schemeで書かれている。6割程度しか理解できなかったが、得たものは多かった。特に4章の超言語的抽象はParenを作るうえで非常に参考になった。

データ構造+アルゴリズム=プログラム

アルバイト時、Javaの文法の本を一冊読み終えた後、職場の先輩に貸していただいた本。悲しことに絶版。

ソーティングアルゴリズム、バックトラックアルゴリズム、PL/0言語のコンパイラ作成等に感銘を受けた。

この業界に進ことに決めたのは、間違いなくこの本のせい。

文法の本終えたばかりのアルバイトに読ませる本だったのかは謎が深い。

ソフトウェア作法

ソフトウエア作成の精神を学んだ。今読み返しても学ぶことが多い。Ratforになれるのに少し時間がかかったが、それ以上に読む価値がある一冊。

Land of Lisp

最初に読んだCommon Lispの本。非常にユーモアにあふれており、随所の挿絵や巻末の漫画も面白かった。

On Lisp

Common Lispの良書。マクロをはじめ、多くのことを学べた。

ハッカーと画家

Lisp系の書物を読み漁っていた時に発見した本。Paul Grahamのエッセイ集で、この業界でない人にも面白く読んでもらえると思う。

ライト、ついてますか

問題解決の本。いかにして問題を解くかよりも砕けていて面白かった。
